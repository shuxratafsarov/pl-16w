## Что делаем

Переносим данные из JSON в Lovable Cloud (БД), добавляем страницу админки с загрузкой Excel и парольным доступом. Публичный дашборд читает из БД (с фолбэком на JSON, если БД пустая).

## База данных

Две таблицы:

- **`weeks`** — `week int PRIMARY KEY`, `period text`, `data jsonb` (целиком JSON недели как сейчас в `week*.json`), `updated_at timestamptz`
- **`monthly`** — `month int PRIMARY KEY`, `data jsonb`, `updated_at timestamptz`

RLS:
- SELECT — публично (любой может читать дашборд)
- INSERT/UPDATE/DELETE — запрещены через клиент. Все записи только через server function, которая проверяет пароль.

## Авторизация

Один общий пароль (как ты и просил), хранится как секрет `ADMIN_PASSWORD` в Lovable Cloud.

- Страница `/admin` — поле ввода пароля
- При вводе пароль уходит на server function, при успехе записывается в `sessionStorage` (чтобы не вводить каждый раз в рамках сессии)
- Все мутации (`upsertWeek`, `uploadExcel`, `deleteWeek`) идут через server functions, которые сами проверяют пароль из заголовка/тела. Без проверки — ничего записать нельзя.

## Страница `/admin`

Три блока:
1. **Загрузить Excel** — кнопка "Загрузить .xlsx", парсится на сервере (библиотека `xlsx`/SheetJS), все недели и месячные данные перезаписываются в БД. Прогоняются те же правила: маркеры M1–M4, `is_auto` для MKO (А), `is_hk_danger` для HONGKONG, пропорциональное распределение CAINIAO по странам.
2. **Список недель** — таблица всех недель с кнопкой "Удалить".
3. **Ручное редактирование одной недели** (опционально, если будет нужно) — JSON-редактор для точечных правок.

## Дашборд

Главная страница:
- Заменяет `import.meta.glob('@/data/week*.json')` на загрузку из БД через server function
- Если БД пустая (ещё не загружали Excel) — фолбэк на текущие JSON файлы, чтобы ничего не сломалось
- Структура данных не меняется → компоненты `OverviewAnalytics`, `MarkersBlock`, `MonthlyView` и т.д. работают без правок

## Что остаётся как есть

- JSON файлы в `src/data/` — остаются как фолбэк и эталон для первой загрузки
- Скрипты валидации (`validate-data.mjs`, `validate_markers.py`) — продолжают работать против JSON, но больше не блокируют билд (или мигрируют в server-side проверку при загрузке Excel)

## Шаги реализации

1. Миграция: создать таблицы `weeks` и `monthly` с RLS (только SELECT публично)
2. Установить `xlsx` (SheetJS) для парсинга Excel в server function
3. Server functions: `listWeeks`, `getWeek`, `verifyPassword`, `uploadExcel`, `deleteWeek`, `upsertWeek`
4. Сидинг: одноразово залить текущие 19 недель + monthly из JSON в БД (через миграцию или на первой загрузке `/admin`)
5. Страница `/admin` с парольным гейтом и формой загрузки Excel
6. Главная страница: переключить чтение на server function с фолбэком на JSON
7. Добавить ссылку на `/admin` в шапку (маленькая иконка-замок)

## Технические детали

- Парсинг Excel в Worker runtime: библиотека `xlsx` (SheetJS) работает в Cloudflare Workers, native deps не нужны
- Пароль проверяется через `timingSafeEqual` против `process.env.ADMIN_PASSWORD`
- После загрузки Excel — транзакция: удалить все строки и вставить заново, чтобы не остались "осиротевшие" недели
